TypeORM

TypeORM - data-mapper ORM for TypeScript and JavaScript (ES7, ES6, ES5). Supports MySQL, PostgreSQL, MariaDB, SQLite, MS SQL Server, Oracle, WebSQL databases. Works in Node.JS and Browser.

Entity Manager

Using EntityManager you can manage (insert/update/delete/fetch) any entity of your connection. EntityManager can be obtained from your connection.

  • hasId(entity: Entity): boolean hasId(target: string, entity: Entity): boolean

    Checks if entity has an id. If entity contains compose ids, then it checks them all.

  • createQueryBuilder(target: Function|string, alias: string): QueryBuilder<Entity>

    Creates a new query builder that can be used to build a sql query.

  • create(target: Function|string): Entity

    Creates a new entity instance.

  • create(target: Function|string, plainObject: Object): Entity

    Creates a new entity instance and copies all entity properties from this object into a new entity. Note that it copies only properties that present in entity schema.

  • create(target: Function|string, plainObjects: Object[]): Entity[]

    Creates a new entities and copies all entity properties from given objects into their new entities. Note that it copies only properties that present in entity schema.

  • preload(target: Function|string, object: Object): Promise<Entity>

    Creates a new entity from the given plan javascript object. If entity already exist in the database, then it loads it (and everything related to it), replaces all values with the new ones from the given object and returns this new entity. This new entity is actually a loaded from the db entity with all properties replaced from the new object.

  • merge(target: Function|string, ...objects: ObjectLiteral[]): Entity

    Merges multiple entities (or entity-like objects) into a one new entity.

  • persist(entity: Entity): Promise<Entity>
    persist(entities: Entity[]): Promise<Entity[]>
    persist(target: Function|string, entity: Entity): Promise<Entity>
    persist(target: Function|string, entities: Entity[]): Promise<Entity[]>

    Persists (saves) all given entities in the database. If entities do not exist in the database then inserts, otherwise updates.

  • remove(entity: Entity): Promise<Entity>
    remove(entities: Entity[]): Promise<Entity[]> remove(target: Function|string, entity: Entity): Promise<Entity>
    remove(target: Function|string, entities: Entity[]): Promise<Entity[]>

    Removes given entity/entities from the database.

  • find(target: Function|string): Promise<Entity[]>
    find(target: Function|string, conditions: ObjectLiteral): Promise<Entity[]>
    find(target: Function|string, options: FindOptions): Promise<Entity[]>
    find(target: Function|string, conditions: ObjectLiteral, options: FindOptions): Promise<Entity[]>

    Finds entities that match given conditions and/or find options.

  • findAndCount(target: Function|string): Promise<[ Entity[], number ]>
    findAndCount(target: Function|string, conditions: ObjectLiteral): Promise<[ Entity[], number ]>
    findAndCount(target: Function|string, options: FindOptions): Promise<[ Entity[], number ]>
    findAndCount(target: Function|string, conditions: ObjectLiteral, options: FindOptions): Promise<[ Entity[], number ]>

    Finds entities that match given conditions. Also counts all entities that match given conditions, but ignores pagination settings (maxResults, firstResult) options.

  • findOne(target: Function|string): Promise<Entity>
    findOne(target: Function|string, conditions: ObjectLiteral): Promise<Entity>
    findOne(target: Function|string, options: FindOptions): Promise<Entity>
    findOne(target: Function|string, conditions: ObjectLiteral, options: FindOptions): Promise<Entity>

    Finds first entity that matches given conditions.

  • findOneById(target: Function|string, id: any, options?: FindOptions): Promise<Entity>

    Finds entity with given id. Optionally find options can be applied.

  • query(query: string): Promise<any>

    Executes a raw SQL query and returns a raw database results.

  • transaction(cb: (repository: Repository<Entity>) => Promise<any>|any): Promise<any>

    Wraps given function execution (and all operations made there) in a transaction. All database operations must be executed using provided repository.

  • getRepository<Entity>(target: Function|string): Repository<Entity>

    Gets repository for the given entity class. If single database connection mode is used, then repository is obtained from the repository aggregator, where each repository is individually created for this entity manager. When single database connection is not used, repository is being obtained from the connection.

  • getTreeRepository<Entity>(target: Function|string): TreeRepository<Entity>

    Gets tree repository for the given entity class. If single database connection mode is used, then repository is obtained from the repository aggregator, where each repository is individually created for this entity manager. When single database connection is not used, repository is being obtained from the connection.

  • getSpecificRepository<Entity>(target: Function|string): SpecificRepository<Entity>

    Gets specific repository for the given entity class. If single database connection mode is used, then repository is obtained from the repository aggregator, where each repository is individually created for this entity manager. When single database connection is not used, repository is being obtained from the connection.

  • release(): Promise<void>

    Releases all resources used by entity manager. This is used when entity manager is created with a single query runner, and this single query runner needs to be released after job with entity manager is done.
